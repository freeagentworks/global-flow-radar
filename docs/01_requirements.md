# 01. 要求定義書 - Global Flow Radar

## 1. プロジェクト概要

### 1.1 プロダクト名
**Global Flow Radar**（グローバルフローレーダー）

### 1.2 ビジョン
世界の投資マネーの流れを「見える化」し、プロの機関投資家しかアクセスできなかったグローバル資金フロー情報を、個人投資家にも提供する。

### 1.3 ミッション
- 資金の流出元と流入先を多層的に検知する
- 直感的なビジュアルで瞬時に状況を把握できるUIを提供する
- 無料で始められるプラットフォームとして広くアクセスを開放する

---

## 2. ユーザー要件

### 2.1 ターゲットユーザー

| ユーザー区分 | 用途 | 優先度 |
|---|---|---|
| 個人トレーダー | 自身のトレード判断・ポジション構築の参考 | ★★★ |
| 一般投資家 | ポートフォリオ方針の参考、マクロトレンド把握 | ★★★ |
| セミナー講師 / 教育者 | デモツールとして資金フローを説明 | ★★☆ |

### 2.2 ユーザーストーリー

**個人トレーダーとして：**
- 世界の資金がどのセクター・地域に流入しているか日次で確認したい
- 急激な資金移動が発生した際にアラートを受け取りたい
- 過去の資金フローパターンと現在を比較して、トレンド転換を察知したい

**一般投資家として：**
- 自分のポートフォリオが資金流入トレンドに乗っているか確認したい
- 世界地図上で直感的に資金の流れを理解したい

**セミナー講師として：**
- リアルタイムのデータを使って、受講者に資金フローの概念を説明したい
- 過去の重大イベント時の資金フローを再生・表示したい

## 3. 機能要件

### 3.1 コア機能（MVP）

#### F-001: グローバル資金フローマップ
- **概要**: 世界地図上に資金の流出入を矢印/サンキー図で表示
- **粒度**: 国・地域レベル（米国、日本、欧州、中国、新興国等）
- **データ**: ETF資金フロー、為替フロー、債券市場フローから推計
- **更新頻度**: 日次（前日終値ベース）

#### F-002: セクターローテーション・ダッシュボード
- **概要**: セクター間の資金移動をリアルタイムダッシュボードで表示
- **対象セクター**: GICS 11セクター（テクノロジー、ヘルスケア、金融、エネルギー等）
- **表示形式**: ヒートマップ、バブルチャート、フローダイアグラム
- **比較機能**: 期間比較（1W / 1M / 3M / 6M / 1Y）

#### F-003: 資産クラス間フロー分析
- **概要**: 株式 ↔ 債券 ↔ コモディティ ↔ 暗号資産 ↔ 現金 間の資金移動
- **指標**: 各資産クラスのETF資金フロー集計、利回り比較
- **表示**: サンキー図、積み上げエリアチャート

#### F-004: 個別銘柄・ETFフロートラッカー
- **概要**: 主要ETF・個別銘柄の資金流出入を追跡
- **対象**: SPY, QQQ, EEM, GLD, TLT 等主要ETF + ユーザー指定銘柄
- **データ**: 出来高分析、資金フロー推計、機関投資家ポジション
- **機能**: ランキング表示（流入Top / 流出Top）

#### F-005: 時系列比較・ヒストリカル分析
- **概要**: 過去の資金フローデータを遡って比較分析
- **期間**: 最大5年分のヒストリカルデータ
- **機能**: 期間オーバーレイ、イベントマーカー（リーマン、コロナ等）
- **表示**: マルチラインチャート、差分ヒートマップ

#### F-006: アラート・異常検知
- **概要**: 急激な資金移動パターンを検知して通知
- **検知ロジック**: 標準偏差ベースの閾値 + トレンド乖離
- **通知手段**: アプリ内通知（MVP）、メール/LINE（Phase 2）
- **カスタマイズ**: ユーザーが閾値・対象を設定可能

### 3.2 Phase 2 機能（MVP後）

#### F-007: AI資金フロー解説（LLM連携）
- **概要**: 日次の資金フロー変動をLLMが自然言語で解説
- **内容**: 「本日は米国テック→日本バリューへの資金ローテーションが顕著」等
- **ソース**: ニュース/SNSの感情分析結果と組み合わせ
- **ステータス**: TBD - AI/ML活用範囲は要検討

#### F-008: ニュース/SNS感情分析連動
- **概要**: ニュースヘッドラインやSNS投稿の感情スコアと資金フローを紐づけ
- **データソース**: ニュースAPI、Twitter/X API、Reddit
- **表示**: 感情スコアと資金フロー相関チャート
- **ステータス**: TBD

#### F-009: ML資金フロー予測
- **概要**: 過去パターンから次の資金フロートレンドを予測
- **手法**: LSTM / Transformer ベースの時系列予測
- **活用**: RTX 5090 GPU での学習パイプライン
- **ステータス**: TBD

#### F-010: ユーザー認証・パーソナライズ
- **概要**: ログイン機能、ウォッチリスト、アラート設定保存
- **認証**: NextAuth.js（Google / GitHub OAuth）
- **ステータス**: TBD

#### F-011: ダッシュボードカスタマイズ
- **概要**: ウィジェット配置のドラッグ&ドロップカスタマイズ
- **ステータス**: TBD

#### F-012: SNSシェア機能
- **概要**: チャート・分析結果のSNSシェア、OGP画像自動生成
- **ステータス**: TBD

## 4. 非機能要件

### 4.1 パフォーマンス
- ページ初期ロード: 3秒以内
- ダッシュボードデータ更新: 2秒以内
- 地図レンダリング: 60fps維持
- 同時接続数: MVP時100ユーザー想定

### 4.2 可用性
- 稼働率目標: 99.5%（MVP）
- データ収集バッチ: 失敗時自動リトライ（3回）
- グレースフルデグラデーション: 一部データ欠損時も表示継続

### 4.3 セキュリティ
- HTTPS必須
- APIキーはサーバーサイドのみ（クライアント露出なし）
- レートリミット実装（API保護）

### 4.4 スケーラビリティ
- 水平スケール可能なアーキテクチャ
- データベースパーティショニング（時系列データ）
- CDN活用（静的アセット）

### 4.5 UI言語
- **デフォルト**: 日本語（Japanese）
- **切替対応**: 日本語 / 英語（JP / EN トグル）
- next-intl による i18n をMVPから実装

## 5. データ要件

### 5.1 データソース（詳細は03_data-sources.md参照）

| データ種別 | ソース | 更新頻度 | コスト |
|---|---|---|---|
| ETF資金フロー | Yahoo Finance API / ETF.com | 日次 | 無料 |
| 先物ポジション | CFTC COTレポート | 週次 | 無料 |
| マクロ経済指標 | FRED API | 日次〜月次 | 無料 |
| 株価・出来高 | Yahoo Finance API | 日次 | 無料 |
| 為替レート | FRED / Open Exchange Rates | 日次 | 無料 |
| ニュース/SNS | NewsAPI / Twitter API | 日次 | 無料〜低コスト |

### 5.2 データ保持期間
- リアルタイムデータ: 直近1年（高精度）
- ヒストリカルデータ: 最大5年（日次集約）
- アーカイブ: 5年超は月次集約で保持

## 6. 制約事項

### 6.1 技術的制約
- 無料APIのレートリミット（Yahoo Finance: 2,000リクエスト/時間）
- COTレポートは週次更新（火曜基準、金曜発表）のため日次補完が必要
- ETF資金フローの正確なデータは15分遅延あり

### 6.2 ビジネス制約
- 初期コスト最小化（無料APIベース）
- MVP 2-3ヶ月でのリリース
- 1人開発体制（AI支援あり）

### 6.3 法的制約
- 金融商品取引法に基づく免責表示（投資助言ではない旨）
- データ再配布に関するAPI利用規約の遵守
- 個人情報保護法対応（ユーザー認証実装時）
